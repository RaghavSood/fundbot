Documentation

GiveWei is a Telegram bot that makes it easy to top up and initialize new crypto addresses with funds. Deposit USDC on Base or Avalanche, then swap to hundreds of assets across any supported chain — all from a single Telegram command. 29+ assets work instantly out of the box, and any other token is automatically looked up and matched against provider liquidity.

How It Works

GiveWei holds USDC on two EVM chains — Base and Avalanche — and uses swap providers to convert it into whatever asset you need. The flow is:

  1. The bot derives an EVM wallet for you (or your group chat) from a BIP39 mnemonic.
  2. You deposit USDC to that wallet on Base or Avalanche.
  3. When you run /topup, the bot finds the best swap route, executes the trade from USDC, and delivers the target asset to any address you specify.

The bot automatically selects which chain to swap from based on where you have sufficient USDC balance. If both chains have enough, it picks the route with the best price.

Commands

Command Description
/start Show welcome message and command list.
/address Display your deposit wallet address.
/balance Show USDC and native token balances on Base and Avalanche. Triggers automatic gas refill if native balance is low.
/quote <addr> <$> <CHAIN.ASSET> [routing] Get a swap quote without executing. Shows provider, expected output, and source chain.
/topup <addr> <$> <CHAIN.ASSET> [routing] Execute a swap. Finds the best route, submits the transaction, and tracks it to completion.
/status <id> Check the status of a pending top-up by its short ID.

Examples

/topup 0xABC...123 10 BTC.BTC — Swap $10 USDC into BTC, best provider
/topup 0xABC...123 5 ETH.ETH thorchain — Force Thorchain for the swap
/quote 0xABC...123 20 SOL.SOL private — Quote using only private/custodial providers
/topup 0xABC...123 5 BASE.CBBTC — Token not built-in; bot looks it up, asks you to confirm, then swaps

Routing Hints

Both /quote and /topup accept an optional routing hint as the last argument. This controls which providers are used for the swap.

Hint Type Behavior
(none) default Best price across all available providers.
thorchain provider Force Thorchain. Fails if the pair isn't supported.
simpleswap provider Force SimpleSwap. Fails if the pair isn't supported.
near provider Force Near Intents. Fails if the pair isn't supported.
dex category Only use DEX providers (Thorchain and Near Intents). Non-custodial.
houdini provider Force Houdini Swap (CEX-routed, $50 min). Fails if the pair isn't supported.
hxmr provider Houdini Swap via anonymous XMR routing ($50 min). Not included in automatic routing — must be explicitly requested.
private category Only use private/custodial providers (SimpleSwap and Houdini Swap).

Swap Providers

TC

Thorchain

DEX · Non-custodial

Cross-chain DEX that swaps via on-chain liquidity pools. Your USDC is approved to the Thorchain router contract and swapped in a single transaction. Non-custodial — funds go directly from your wallet to the protocol.

SS

SimpleSwap

Private · Custodial

Custodial exchange aggregator. The bot creates an exchange via SimpleSwap's API, then sends USDC to a deposit address. SimpleSwap handles the swap and delivers the output. 29+ built-in assets including BTC, ETH, SOL, DOGE, XRP, ADA, DOT, ATOM, and more.

NI

Near Intents

DEX · Intent-based

Intent-based cross-chain swaps powered by the NEAR protocol. The bot requests a quote via the 1Click API, deposits USDC to a one-time address, and solvers compete to fill the order at the best rate. Supports BTC, ETH, SOL, AVAX, and 10+ other assets. Track swaps at explorer.near-intents.org.

HS

Houdini Swap

Private · Custodial

Custodial exchange aggregator with CEX routing. The bot creates an exchange via Houdini's partner API, then sends USDC to a deposit address. Houdini routes the swap through its network of exchange partners and delivers the output. Supports BTC, ETH, SOL, AVAX, ADA, DOT, DOGE, and 20+ other assets.

Dynamic Token Discovery

If you request a token that isn't in the built-in list, GiveWei will automatically try to find it. The process works like this:

  1. The bot searches CoinGecko for your token symbol and picks the best match by market cap.
  2. It fetches the token's contract addresses across chains and matches them against Thorchain liquidity pools, Near Intents token list, the SimpleSwap static catalog, and Houdini Swap.
  3. If at least one provider supports the token, the bot shows you the resolved token name, contract address, and available providers.
  4. You tap Confirm to proceed — or Cancel if it's not the right token.

This means you can swap tokens like BASE.CBBTC, ETH.LINK, or ETH.UNI — even though they're not in the built-in list — as long as a provider has liquidity for them.

CoinGecko results are cached for 1 hour and provider token lists for 10 minutes. Symbol matching is case-insensitive. The confirmation step has a 5-minute expiry.

Source Chains

GiveWei sources USDC from two chains. You can deposit to either — the bot picks the best one at swap time.

B

Base

USDC on Base (Coinbase L2). Low fees, fast confirmations. USDC contract: 0x833589fCD6eDb6E08f4c7C32D4f71b54bdA02913

A

Avalanche

USDC on Avalanche C-Chain. Sub-second finality. USDC contract: 0xB97EF9Ef8734C71904D8002F8B6BC66Dd9c48a6E

Automatic Gas Refill

Swaps on Base and Avalanche require a small amount of native gas tokens (ETH and AVAX respectively) to pay transaction fees. GiveWei handles this automatically.

When you check your /balance and a chain's native balance is below ~$1 worth, the bot will swap $5 of USDC into the native token via CoW Protocol (a gasless DEX that uses EIP-2612 permits, so it works even with zero gas balance).

Gas refill orders expire after 3 minutes. If the order doesn't fill in time, you'll get a notification — just check /balance again to retry.

Single vs Multi Mode

Single mode: One shared wallet (derivation index 0). All users share the same address and USDC balance. Best for personal use or small teams.

Multi mode: Each user and each group chat gets a unique wallet derived from the same mnemonic. Users in DMs get their own address; group chats share a group-specific address. Derivation indices are assigned sequentially and never collide.

Self-Hosting

GiveWei is fully open source and designed to be self-hosted. You control the mnemonic, the config, and the deployment.

  1. Clone the repository and build with go build ./cmd/fundbot.
  2. Create a config.json based on config.example.json. You'll need a Telegram bot token, a BIP39 mnemonic, and RPC endpoints for Base and Avalanche.
  3. Run database migrations: goose -dir db/migrations sqlite3 fundbot.db up
  4. Start the bot: ./fundbot -config config.json

The bot runs a web dashboard on the configured HTTP port with live stats, charts, and an admin panel.
